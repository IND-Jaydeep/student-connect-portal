Testing Approach

Table 5.2 Testing Approach for Other Modules

Module Testing Approach
Grievance Management 1. Submit grievances with valid descriptions and verify they are saved and routed to admin.
2. Submit grievances with empty details and check for validation errors.
3. Admin updates grievance status and verify the student can see the updated status.
Attendance Tracking 1. Mark attendance for a student for a specific date and verify it is recorded.
2. Attempt to mark attendance for a future date and check for error handling.
3. Generate attendance statistics and ensure calculations are accurate.
Chatbot 1. Input common queries in text format and verify relevant responses.
2. Submit unrecognized queries and verify escalation to human/admin intervention or default fallback response.
Notice Board 1. Admin creates a notice and verifies it is visible to the intended user roles (e.g., all students).
2. Test boundary cases like very long notice descriptions and missing titles.
